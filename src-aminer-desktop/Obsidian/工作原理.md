# 工作原理

## 整体架构

```
标注平台 (annot.aminer.cn)
     ↓ AMiner API (prompts, questions, images)
     
aminer-desktop.exe (localhost:9800)
     ↕ HTTP
扩展 background.js (Service Worker)
     ↕ chrome.runtime port
扩展 monitor.js (内容脚本)
     ↓ 替换 img.src
标注页面 (图片从 localhost 加载)
```

## 核心流程

### 阶段一：检测

```
1. monitor-main.js 注入页面主世界
2. 拦截 XMLHttpRequest.prototype.send
3. 检测到 POST/PUT/PATCH → /responses (标注提交)
4. 读取页面"第X题/Y"
5. dispatchEvent('aminer-monitor:event')

6. monitor.js 接收事件
7. chrome.runtime port → background.js
```

### 阶段二：预取

```
8. background.js 收到提交事件
9. ensureGroup(当前题号 + 1)
10. 判断当前缓存是否覆盖当前题号
    ├─ 覆盖且够 50 张 → 不做操作
    └─ 不足或超出范围 → rebuildGroup

11. rebuildGroup:
    a. clearCache() — 删除旧缓存
    b. nextPendingPrompts(题号):
       - page = ceil(题号/20)
       - api('/api/prompts?page=' + page) — 获取当前页 20 条
       - 筛选 state≠1 (未标注) 的题目
       - 继续下一页，直到凑满 50 条或到第 100 页
    c. 遍历每个题目:
       - api('/api/question?id=' + prompt_id) — 获取图片 URL
       - api('/api/cache', {url, promptId, questionNum}) — 下载图片
```

### 阶段三：替换

```
12. background.js 每 1 秒:
    - api('/api/images') — 获取最新缓存列表
    - port.postMessage({type:'cache-list', images})

13. monitor.js 接收缓存列表
14. 遍历页面所有 <img>、<source>、<video>、style:
    - 如 src 为 OSS URL
    - 查找缓存列表中对应图片
    - 替换为 http://127.0.0.1:9800/api/image/{hash}

15. 图片从本地磁盘加载（毫秒级）
```

## 跨页策略

```
nextPendingPrompts(startQuestion):
  page = ceil(startQuestion / 20)
  startIdx = startQuestion - (page-1)*20 - 1
  
  while items < 50 AND page <= 100:
    data = api('/api/prompts?page=' + page)
    for i = startIdx to 19:
      if prompts[i].state != 1:
        items.push({prompt, questionNum})
    page++
    startIdx = 0          # 后续页从第 0 个开始
  
  return items[0..49]
```

## 缓存生命周期

```
          标注提交
             ↓
      ensureGroup(q+1)
             ↓
    ┌─ 缓存覆盖当前题? ──┐
    ↓是                  ↓否
   充足?             rebuildGroup
    ↓是 ↓否               ↓
   不动 补充         清除旧缓存 → 从当前题拉 50 张
             ↓
   下次提交，重复检查
```

## 通信隔离

| 组件 | 运行环境 | 通信方式 | 安全性 |
|------|----------|----------|--------|
| monitor-main.js | 页面主世界 | CustomEvent | 只发事件，不访问其他扩展 |
| monitor.js | 扩展隔离世界 | chrome.runtime port | 浏览器 IPC |
| background.js | Service Worker | fetch localhost | 无 CORS 限制 |
| aminer-desktop.exe | 本机进程 | HTTP Server | 仅监听 127.0.0.1 |

## 关键设计决策

| 决策 | 原因 |
|------|------|
| 图片 URL 替换而非拦截请求 | 避免破坏页面原有逻辑 |
| Service Worker 发 HTTP 而非内容脚本 | 内容脚本受页面 CORS 限制 |
| 50 张一组而非逐张 | 减少 API 调用，提升预取效率 |
| 按页批量获取 prompt | 避免跨页时逐题请求的 401/失败 |
| 本地 exe 而非纯扩展 | 磁盘缓存无上限，不占浏览器存储 |
