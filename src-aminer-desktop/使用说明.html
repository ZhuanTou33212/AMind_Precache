📦 AMiner 预缓存系统

版本 2.0 | 2026-06-18

项目简介

本系统为 AMiner 标注平台提供图片预缓存功能。将 OSS 云端图片提前下载到本地，标注页面直接从本机加载图片，实现秒开效果。

文件结构

AMiner预缓存系统/
├── aminer-desktop.exe   ← 桌面应用（双击启动）
├── 使用说明.html          ← 本文档
├── extension/            ← Chrome/Edge 扩展
│   ├── manifest.json
│   ├── background.js     ← Service Worker（核心预缓存逻辑）
│   ├── monitor.js        ← 内容脚本（图片替换）
│   └── monitor-main.js   ← 主世界脚本（提交检测）
└── data/                 ← 缓存数据（自动生成）
    ├── config.json
    ├── cache.db          ← SQLite 数据库
    └── images/           ← 图片文件

快速开始

1 启动桌面应用
双击 aminer-desktop.exe，黑色终端窗口不要关闭。
2 加载扩展
打开 Edge → edge://extensions/ → 开启「开发人员模式」→「加载解压缩的扩展」→ 选择 extension/ 文件夹。
3 获取 Token
打开标注页面 → F12 → Console → 输入 允许粘贴 回车 → 粘贴：
var _s=XMLHttpRequest.prototype.setRequestHeader;
XMLHttpRequest.prototype.setRequestHeader=function(k,v){
  if(k.toLowerCase()==='authorization') console.log('TOKEN:',v.replace('Bearer ',''));
  return _s.apply(this,arguments);
}

点标注页任意数字键提交一次，Console 出现 TOKEN: eyJhbGci...，复制这段。

4 配置
打开 http://127.0.0.1:9800/ → 粘贴 Token → 点「💾 保存」。
5 开始使用
正常标注即可。系统自动预取 50 张图片、替换页面图片 URL 为本地地址。

工作原理

组件作用
exe本地服务器 (9800端口) + SQLite + 图片存储
background.js调用 AMiner API 获取题目 → 下载图片 → 管理缓存
monitor.js接收缓存列表 → 替换页面 img.src 为 localhost
monitor-main.js检测标注提交 → 触发缓存更新
⚠️ 注意
• exe 必须保持运行
• Token 约 2 小时过期，过期需重新获取
• 缓存容量 50 张，自动轮替
• 支持跨页获取（自动翻页）
