## 2026-06-21 预标注窗口与单题标签隔离

### 本次确认

当前后端写入标签不是给所有图片统一打标签。

`/api/image-labels` 会根据以下标识定位一条缓存图片记录：

```text
hash
promptId
questionNum
```

只要请求中带有当前题号或图片 hash，就只会更新对应图片。

### 本次修正

扩展端原先使用全局临时标签状态，切换题号时有可能沿用上一题的临时选择。

已改为：

```text
labelsByQuestion[questionNum] = labels
```

每个题号独立保存草稿，快捷键和 Backspace 只影响当前题号。

题号识别已增加兼容格式：

```text
第1题
题号:1
当前第1
1/2000
Question1
```

### 新增 EXE 页面机制

在 EXE 内嵌页面新增了“预标注窗口”：

1. 实时轮询 `/api/monitor-state` 读取当前题号。
2. 当前题号为 1 时，从本地缓存中查找 `questionNum=1` 的图片并放入预标注窗口。
3. 点击标签后只修改当前预标注窗口内图片的临时标签。
4. 点击“保存标注并进入下一张”后，写入 `/api/image-labels`。
5. 已保存图片进入“已标注数据缓存层”，未保存图片留在“未标注数据缓存层”。
6. 保存后自动从未标注层选择当前题号之后的下一张图片。

### 新增回看能力

新增两个按钮：

```text
返回上一张已标注
选择已标注图片
```

这两个入口会把已标注图片重新放回预标注窗口，允许重新选择标签并覆盖保存。

### 2026-06-21 v1.13 缓存槽工作流

新增功能：

```text
1. 预标注窗口新增放大/缩回预览槽。
2. 放大预览支持自定义按键，默认 Z。
3. 新增未标注缓存槽。
4. 新增预确定缓存槽。
5. 新增已完成标注缓存槽。
```

槽位状态定义：

```text
未标注缓存槽 = label_status pending，且没有 labelText
预确定缓存槽 = label_status labeled，且有 labelText
已完成标注缓存槽 = label_status submitted
```

未标注缓存槽操作：

```text
顶部标签按钮选择标签 -> 左键选择多张图片 -> 右键或确认按钮 -> 批量写入标签 -> 进入预确定缓存槽
```

预确定缓存槽操作：

```text
可重新选择顶部标签按钮 -> 左键选择多张图片 -> 右键或确认按钮 -> 提交所选到已完成标注缓存槽
也可以点击“全部提交”提交整个预确定槽。
```

预标注窗口切换：

```text
可以从未标注缓存槽切换到第一张未标注图片。
可以从预确定缓存槽切换到第一张预确定图片。
```

验证结果：

```text
pending -> labeled -> submitted 数据层流转通过。
5 张占位缓存已在测试后重置为 pending。
```

### 2026-06-21 v1.15 已完成槽网页提交按钮

在“已完成标注缓存槽”新增：

```text

全部提交到网页
取消选择
```

本地状态流转：

```text
submitted / cloud_submitted / submit_failed -> 仍显示在已完成标注缓存槽
点击提交按钮 -> label_status = submit_requested
```

网页扩展处理：

```text
monitor-quicklabel.js 轮询 cache-list
如果当前网页题号对应图片状态为 submit_requested：
1. 读取本地 labels
2. 扫描网页 label/button/radio/checkbox
3. 按 label 和 aliases 匹配网页选项
4. 点击对应选项
5. 尝试点击“提交/确认/保存”等按钮
6. 如找不到按钮，则尝试派发 Ctrl+Enter 和 Enter
7. 成功后标记 cloud_submitted
8. 失败后标记 submit_failed
```
