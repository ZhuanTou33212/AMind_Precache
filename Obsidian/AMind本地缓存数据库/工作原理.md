# 工作原理

## 三层架构

```
第 1 层: 页面主世界 (monitor-main.js)
  ↕ CustomEvent
第 2 层: 扩展隔离世界 (monitor.js)
  ↕ chrome.runtime port
第 3 层: Service Worker (background.js)
  ↕ HTTP localhost:9800
第 4 层: 桌面应用 (aminer-desktop.exe)
  ↕ HTTPS
AMiner API
```

## 阶段一: 检测标注提交

```
1. monitor-main.js 注入页面主世界
   使用 manifest.json 中 world: "MAIN" 配置

2. 拦截 XMLHttpRequest.prototype.send
   保存原始方法引用 → 包装新方法 → 在 loadend 事件中检测

3. 提交识别规则:
   HTTP 方法: POST | PUT | PATCH
   状态码: 200-299
   URL 匹配: /\/(label|annot|submit|save|answer|mark|review|commit|responses)\b/i

4. 读取页面文本 "第X题/Y"
   正则: /第(\d+)题\/(\d+)/

5. dispatchEvent(new CustomEvent('aminer-monitor:event', {detail}))
   主世界 → 隔离世界 (同页面不同 JS 上下文)
```

## 阶段二: 预取图片

```
6. monitor.js 接收事件 → chrome.runtime.connect port → background.js

7. background.js ensureGroup(当前题号):
   
   step 1: api('/api/images') 获取已有缓存列表
   step 2: 提取所有 questionNum 找 min/max
   step 3: 判断条件:
     - 缓存为空 OR 当前题 < min OR 当前题 > max → outsideCurrentGroup
   step 4: outsideCurrentGroup 为 true → rebuildGroup(当前题)

8. rebuildGroup:
   a. api('/api/images', DELETE) — 清空全部旧缓存
   b. nextPendingPrompts(当前题号) — 获取待缓存列表
   c. 遍历列表: cachePrompt(item) — 下载每张图片

9. nextPendingPrompts(q):
   page = ceil(q / 20)         # 从哪页开始
   startIdx = q - (page-1)*20 - 1  # 页内偏移
   
   limit = cacheSize 或默认 50
   
   while items < limit AND page ≤ 100:
     data = api('/api/prompts?page=' + page)
     从 startIdx 遍历到 19:
       如果 state ≠ 1 → items.push({prompt, questionNum})
     page++, startIdx = 0      # 后续页从头开始
   
   return items[0..limit-1]

10. cachePrompt(item):
    detail = api('/api/question?id=' + item.prompt.prompt_id)
    从 response 正则提取 OSS URL
    api('/api/cache', POST {url, promptId, questionNum})
    → exe 下载图片到磁盘 → 存入 SQLite
```

## 阶段三: 图片替换

```
11. background.js 每 1 秒:
    images = api('/api/images')     # 获取最新缓存列表
    port.postMessage({type:'cache-list', images})

12. monitor.js 接收缓存列表:
    构建 Map: objectKey(url) → {hash, url, ...}
    objectKey = URL.origin + URL.pathname  (去掉查询参数)

13. replaceImages() 遍历页面:
    - document.querySelectorAll('img[src]')
    - document.querySelectorAll('source[srcset], img[srcset]')
    - document.querySelectorAll('[style*="mm-group-image"]')
    
    如果 src 匹配 OSS 域名且在缓存中:
      → 替换为 http://127.0.0.1:9800/api/image/{hash}
      → 保存原始 URL 到 dataset.aminerOriginalSrc

14. MutationObserver 监听新增元素
    自动对新出现的 img 执行相同替换逻辑
```

## 跨页缓存算法

```
假设当前题号 = 1973
page = ceil(1973/20) = 99
startIdx = 1973 - 98*20 - 1 = 12

第 99 页: 取 prompt[12..19] 共 8 个未标注 → item count = 8
第 100 页: 取 prompt[0..19] 共 20 个未标注 → item count = 28
总计: 28 个 (如果 state≠1 的不足则停在 <= 50)
```

## 通信安全性

| 链路 | 机制 | 隔离保证 |
|------|------|----------|
| 主世界→隔离世界 | CustomEvent | 不同 JS 上下文 |
| 隔离世界→SW | chrome.runtime port | 浏览器 IPC |
| SW→隔离世界 | port.postMessage | 浏览器 IPC |
| SW→exe | HTTP localhost | 127.0.0.1 仅本机 |
| exe→AMiner | HTTPS | TLS 加密 |

## 为什么这样设计

| 决策 | 原因 |
|------|------|
| 图片 URL 替换 (不改 DOM) | 避免破坏页面 React/Vue 状态 |
| SW 发 HTTP (非内容脚本) | 内容脚本受页面 CORS/Mixed Content 限制 |
| 默认 50 张一组，可配置 | 减少 API 调用；可按机器性能和网络状况调整 |
| 按页批量 (非逐题) | 避免跨页时逐题 401/超时 |
| 本地 exe 磁盘存储 | 不受浏览器存储配额限制 |
| 纯 Go (无 CGO) | 单文件 exe，无需运行时 |
