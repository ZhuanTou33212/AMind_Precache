# 使用说明

## 安装

### 第 1 步: 启动服务端

双击 `aminer-desktop.exe`，保持黑色终端窗口运行。

> 出现 Windows 防火墙提示请点「允许」。

### 第 2 步: 加载扩展

```
edge://extensions/ → 开发人员模式 → 加载解压缩的扩展
→ 选择 extension/ 文件夹
```

### 第 3 步: 打开标注页面

打开 AMiner 标注页面并保持登录状态。

扩展会自动读取页面登录态，将 `Access-Token`、`taskId`、`startDate` 保存到本地服务，无需手动复制 Token。

### 手动 Token 兜底

如果自动配置失败，再使用旧方法手动获取 Token：

打开标注页面 → F12 → Console → 输入 `允许粘贴` 回车 → 粘贴：

```js
var _s=XMLHttpRequest.prototype.setRequestHeader;
XMLHttpRequest.prototype.setRequestHeader=function(k,v){
  if(k.toLowerCase()==='authorization') console.log('TOKEN:',v.replace('Bearer ',''));
  return _s.apply(this,arguments);
}
```

点任意数字键(1-5)提交标注 → Console 出现 `TOKEN: eyJhbGci...` → 复制这段。

然后打开 `http://127.0.0.1:9800/` → 粘贴 Token → 点「💾 保存」。

### 第 4 步: 检查缓存

打开 `http://127.0.0.1:9800/`，确认配置已写入后即可正常标注。

## 日常使用

**正常标注即可，无需额外操作。** 系统自动：

- 检测当前题目编号
- 从 AMiner API 获取后续若干张未标注题目的图片
- 下载图片到本地磁盘
- 将页面上的 OSS 图片 URL 替换为本地地址

## 缓存数量

默认缓存 **50 张**。如需调整，可编辑 `extension/settings.json`：

```json
{
  "cacheSize": 200
}
```

支持范围: **1-200**。数值越大，首次预取越慢、占用磁盘越多，但跨题时更不容易等待加载。修改后请在浏览器扩展页重新加载扩展，或重启 AMiner 快捷方式。

## 管理界面

`http://127.0.0.1:9800/` 提供：

| 功能 | 说明 |
|------|------|
| 📌 置顶 | 窗口钉在最上层 |
| 💾 保存 | 保存 Token 配置 |
| 📋 加载题目 | 查看题目列表和标注状态 |
| 🚀 预取 | 手动预取图片 |
| 🗑 清空 | 删除全部缓存图片 |

---

## Token 过期

Token 约 **2 小时** 过期。

**现象**: 图片不再缓存、Console 出现 401、缓存数量不增加。

**处理**: 刷新 AMiner 标注页面，扩展会自动重新同步 Token；如果仍失败，再使用“手动 Token 兜底”。

## 故障排查

| 现象 | 检查项 |
|------|--------|
| 图片没替换 | exe 是否运行？扩展是否加载？ |
| 401 错误 | Token 是否过期？ |
| CORS 错误 | 扩展是否刷新？|
| 页面没反应 | 是否在标注页面？URL 是否包含 `label_page_feed`？ |
| exe 启动失败 | 是否已有实例在运行？端口 9800 是否被占用？ |
| 扩展加载失败 | manifest.json 是否完整？文件路径是否正确？ |

## 缓存数据

- **位置**: `data/images/` (图片) + `data/cache.db` (数据库)
- **容量**: 默认 50 张，可通过 `extension/settings.json` 的 `cacheSize` 调整
- **清理**: 删除 `data/` 文件夹即可重置
